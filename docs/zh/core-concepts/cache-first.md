# Cache-First 设计

Halo 的定义性哲学：**将稳定内容与动态内容分离**以最大化缓存命中。

## 问题

每次 LLM API 调用包含：

- **系统提示词** — 模型指令
- **工具定义** — 可用工具的 JSON Schema
- **对话历史** — 用户和助手消息

系统提示词和工具定义在轮次之间很少变化。但在典型实现中，每次请求都重新发送——每次都按全价计费。

## 解决方案

Halo 将消息分成两层：

```
┌──────────────────────────────────┐
│  StablePrefix (已缓存)           │
│  ┌────────────────────────────┐  │
│  │ system: "你是..."          │  │  ← 按缓存价计费
│  │ tools: [get_weather, ...]  │  │     (~74% 折扣)
│  │ fewShots: [...]            │  │
│  └────────────────────────────┘  │
├──────────────────────────────────┤
│  MessageLog (未缓存)             │
│  ┌────────────────────────────┐  │
│  │ user: "天气怎么样？"       │  │  ← 按全价计费
│  │ assistant: "晴天..."       │  │
│  │ user: "巴黎呢？"           │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

## 什么触发缓存未命中

| 操作 | 缓存未命中？ |
|---|---|
| 新用户消息 | 否——历史追加到前缀之后 |
| `agent.clearLog()` | 否——仅重置历史，前缀不变 |
| `agent.addTool(...)` | 是——前缀变化（仅下一次） |
| `agent.setSystem(...)` | 是——前缀变化 |
| `agent.addFewShot(...)` | 是——前缀变化 |

## 测量缓存性能

```ts
console.log(agent.stats.caching?.cacheHitRate);     // e.g. 0.92
console.log(agent.stats.caching?.estimatedSavingsUsd); // e.g. 0.0024
```
