# Halo SDK v1.1

> 状态：已确认 | 更新：2025-07-16

## 原则

- 不破坏 `StablePrefix → MessageLog` 分层
- 不引入 `any`，不改已有公开 API 签名
- 冷启动友好：前缀变更最多触发一次 cache miss

---

## F1. Agent 构造 API 重构

### 变更

`messages: ChatMessage[]` 替代 `system` + `fewShots`。全部进入 StablePrefix，对话历史从第一次 `run()` 进入 MessageLog。`system` / `fewShots` / `setSystem` 标记 `@deprecated`，与 `messages` 互斥。

```ts
const agent = h.agent({
  messages: [
    { role: "system", content: "You are helpful." },
    { role: "user", content: "2+2?" },
    { role: "assistant", content: "4" },
  ],
  tools: { ... },
  skills,
  model: { temperature: 0.7, maxTokens: 4096 },
});
```

新增 `model: ModelConfig` — **不进 prefix，不进缓存指纹**。per-request 可覆盖：`agent.run("hi", { temperature: 0 })`。

### 缓存审计

| 数据 | 进 prefix | 改它 → cache miss |
|---|---|---|
| `messages` | ✅ | 是 |
| `tools` | ✅ | 是 |
| `skills` 元数据 | ✅ | 是 |
| `model.*` | ❌ | 否 |
| per-request `opts.temperature` 等 | ❌ | 否 |

### 实现细节

- `HaloAgentImpl` 构造函数解析 `messages`：提取 `system` 角色消息 → `StablePrefix`，其余消息 → `StablePrefix.fewShots`。若同时传入 `messages` 和 `system`，抛 `AmbiguousPrefixError`。
- `messages` 为空时不报错，prefix 长度为 0，行为等同于无 system prompt 的裸 agent。
- `model` 字段存储为实例属性 `_modelDefaults`，在 `_callModel()` 时 merge 到 adapter 调用的 `options` 中，per-request `opts` 优先级更高。
- `addFewShot` / `removeFewShot` / `setSystem` 保留但 `@deprecated`，内部转换为操作 `messages` 前缀。

### 验收

| # | 标准 |
|---|---|
| F1.1 | `messages` 正确进入 StablePrefix，fingerprint 随 messages 变更 |
| F1.2 | `model.temperature` 变更不触发 cache miss，透传到 adapter |
| F1.3 | `system` + `messages` 同时传入时抛错 |
| F1.4 | per-request `opts.temperature` > agent `model.temperature` |
| F1.5 | `addFewShot` / `setSystem` 仍可用，触发一次 cache miss |

## F2. MCP 支持 → `@halo-sdk/mcp`

### 设计

依赖官方 `@modelcontextprotocol/sdk`（peer dep），薄封装。`tools()` 返回 `Record<string, ToolDefinition>`，`execute()` 自动代理 `client.callTool()`。用户喂给 `h.agent({ tools })` → 进 StablePrefix。

```ts
const server = await createMCPServer({
  transport: { type: "stdio", command: "npx", args: ["-y", "mcp-server"] },
});
const tools = await server.tools();
const agent = h.agent({ messages: [...], tools });
// ...
await server.close();
```

`listResources` / `readResource` / `listPrompts` / `getPrompt` 全部暴露为可选 slot，非核心路径。`serverInfo` / `instructions` 作为元数据暴露。

### 实现细节

- `createMCPServer` 内部：根据 `transport.type` 创建 `StdioClientTransport` 或 `SSEClientTransport`（来自 MCP SDK），连接 → `client.initialize()` → 返回 `MCPServerConnection`。
- `tools()` 调用 `client.listTools()` → 遍历结果 → `definitionToSpec(name, { description, parameters, execute })`。`execute` 闭包捕获 `client` 引用，内部调用 `client.callTool({ name, arguments })`。
- `close()` 调用 `client.close()` + transport 清理。`serverInfo` / `instructions` 在 `initialize()` 阶段缓存。
- SSE transport 支持 `headers` / `authProvider` 透传。若连接失败，`createMCPServer` 抛 `MCPConnectionError`（包装原始错误）。
- 子进程泄漏风险：文档强调 `try/finally`，README 示例用 `using` 语句（TS 5.2+）。

### 验收

| # | 标准 |
|---|---|
| F2.1 | stdio transport：`npx -y @anthropic/mcp-server-filesystem` 工具发现成功 |
| F2.2 | MCP 工具的 `execute()` 正确代理 `client.callTool()` |
| F2.3 | `close()` 后子进程退出（`ps` 验证） |
| F2.4 | SSE transport：连接到 HTTP MCP server 成功 |
| F2.5 | `tools()` 结果喂给 `h.agent({ tools })` 后正常工作 |
| F2.6 | `listResources` / `getPrompt` 返回有效数据 |

## F3. Agent Skills 原生支持 → `@halo-sdk/core`

### 设计

遵循 [agentskills.io](https://agentskills.io) 渐进披露：扫描 `SKILL.md` → name+description 进 system prompt（StablePrefix）→ 运行时 `loadSkill` 工具读 body（MessageLog）。Skill body 跨轮持久，由 `ContextStrategy` 自然截断。

```ts
// 自动发现（@nodeOnly）
const skills = await discoverSkills({ directories: [".halo/skills"] });

// 手动跨平台
const skills = [{ name: "pdf", description: "...", path: "/..." }];

const agent = h.agent({ messages: [...], tools: {...}, skills });
```

### 实现细节

- `discoverSkills`：用 `fs.readdir` 扫描目录 → 每个子目录读 `SKILL.md` → 正则提取 YAML frontmatter（`/^---\n([\s\S]*?)\n---/`）→ 简单 YAML 解析（仅支持 `name` 和 `description` 字符串字段，不依赖完整 YAML 库）→ 去重（首个发现的 skill 胜出，允许项目级覆盖）。
- `loadSkill` 工具自动注册：`HaloAgentImpl` 构造函数中，若 `skills` 非空且长度 > 0，内部生成一个 `ToolDefinition`，name 固定为 `loadSkill`，parameters 为 `{ type: "object", properties: { name: { type: "string" } }, required: ["name"] }`，`execute` 闭包捕获 `skills` 数组，按 name 匹配 → `fs.readFile(skillPath/SKILL.md)` → 去掉 frontmatter → 返回 body。
- Skill 元数据注入 system prompt：在 `StablePrefix` 构造时，若 `skills` 非空，将 `"\n## Available Skills\n" + skills.map(s => `- ${s.name}: ${s.description}`).join('\n')` 追加到 system 消息的 content 末尾。
- `loadSkill` 和 `tools` 中的同名工具冲突：`loadSkill` 是保留名，用户 tools 中不能有 `loadSkill`。若冲突，构造时抛 `ReservedToolNameError`。
- `skills: []` 时不注入元数据也不注册 `loadSkill` 工具。

### 验收

| # | 标准 |
|---|---|
| F3.1 | `discoverSkills` 正确扫描目录并提取 name+description |
| F3.2 | 同名 skill 去重：项目目录覆盖全局目录 |
| F3.3 | `loadSkill` 工具自动注册，不在用户 `tools` 中显式定义 |
| F3.4 | 调用 `loadSkill` 后 body 进 MessageLog，后续 `run()` 可见 |
| F3.5 | 空 `skills` 时不注册 `loadSkill` 工具 |
| F3.6 | 用户 tools 中包含 `loadSkill` 时抛 `ReservedToolNameError` |

## F4. 流式生成 → `streamText()` + `streamAgent()`

### 设计

`streamText()` 是 `run()` 的流式版本 — 带完整工具循环，支持命名回调。旧 `stream()` / `sdkStream()` 保留不动。`Halo` 工厂新增 `streamAgent()` 便捷方法。

**第一个参数多态：**

```ts
// 字符串（普通对话）
const stream = agent.streamText("What's the weather?");

// 消息数组（AI SDK useChat 集成）
const stream = agent.streamText(messages, { ... });
// 内部：最后一条 → 当前输入，之前 → hydrate() → MessageLog
```

**工具循环 + 命名回调：**

```ts
const stream = agent.streamText("What's 2+2 and the weather?", {
  maxSteps: 10,                    // 工具循环上限（默认 10）
  temperature: 0.2,                // per-request 覆盖 model 默认

  // ── 命名回调（AI SDK 对齐）──
  onChunk: ({ chunk }) => {
    // 每个 TurnChunk 到达时触发
    // chunk: { type: "text-delta", delta: "..." }
    //      | { type: "tool-call-ready", index, call }
  },
  onStepFinish: ({ text, toolCalls, usage, step }) => {
    // 每一步工具执行后、模型返回完成时触发
    // step 从 1 开始
  },
  onFinish: ({ text, usage, steps, toolCalls }) => {
    // 全部步骤完成后触发，含累积统计
  },
  onError: (err) => {
    // 任何步骤中出错（网络、解析、工具执行失败等）
  },
});

// ── 事后钩子（动态添加）──
stream.on("tool-call-ready", (call) => { ... });
stream.on("done", (u) => { ... });
```

**完整示例：Next.js App Router + useChat**

```ts
// app/api/chat/route.ts
import { Halo, discoverSkills } from "@halo-sdk/core";
import { DeepSeekAdapter } from "@halo-sdk/adapters";
import { createMCPServer } from "@halo-sdk/mcp";

const h = new Halo({
  adapter: new DeepSeekAdapter({ apiKey: process.env.DEEPSEEK_API_KEY! }),
});
const skills = await discoverSkills({ directories: [".halo/skills"] });
const mcp = await createMCPServer({
  transport: { type: "stdio", command: "npx", args: ["-y", "mcp-server-filesystem"] },
});
const mcpTools = await mcp.tools();

export async function POST(req: Request) {
  const { messages } = await req.json();

  const agent = h.agent({
    messages: [{ role: "system", content: "You are helpful with file access." }],
    tools: mcpTools,
    skills,
    model: { temperature: 0.7 },
  });

  return agent.streamText(messages, {
    onFinish: ({ text, usage, steps }) => {
      console.log(`done: ${steps} steps, ${usage.promptTokens} tokens`);
    },
    onError: (err) => console.error("stream error:", err),
  }).toDataStream();
}
```

### 流示意图

```
agent.streamText("What's 2+2 and the weather?")
  │
  ├─ step 1: adapter.stream() → text-delta* + tool-call-ready: calculator
  │   → auto execute → result "4" → append MessageLog
  │   → onStepFinish({ text: "Let me check...", toolCalls: [...], usage, step: 1 })
  │
  ├─ step 2: adapter.stream() → text-delta* + tool-call-ready: get_weather
  │   → auto execute → result "22°C" → append MessageLog
  │   → onStepFinish({ text: "2+2=4.", toolCalls: [...], usage, step: 2 })
  │
  └─ step 3: adapter.stream() → text-delta* → done
      → onFinish({ text: "2+2=4. Tokyo is 22°C.", usage, steps: 3, toolCalls: [...] })
```

### 实现细节

- `streamText` 内部：`typeof arg === "string"` → `this._log.append({ role:"user", content:arg })`；`Array.isArray(arg)` → `hydrate()` 历史部分，最后一条追加为当前输入。
- 工具循环复用 `_callModelStream()`（新增内部方法），每步调用 `adapter.stream()`，累积 `text-delta`、拦截 `tool-call-ready` → 自动执行 → `_log.append({ role:"tool", ... })` → 下一步。
- 命名回调在构造时注册到 `_listeners`，事后 `stream.on()` 通过 `StreamTextResult._hook()` 追加。两者共享同一 listener map，无重复触发。
- `onFinish` 在 `tool-call-ready` 仅当最后一步无工具调用时触发（工具循环结束）。若 `maxSteps` 耗尽仍有工具，`onFinish` 仍触发但 `toolCalls` 非空。
- `onError` 捕获网络错误、JSON 解析失败、execute 抛异常。不中止循环（单步失败后继续）。
- `StreamTextResult` 内部 lazy 创建 `ReadableStream`，`to*()` 共享同一 source。延迟到消费时才启动 adapter 流（cold observable）。
- `streamAgent(opts)` 语法糖：`h.agent(opts)` → `agent.streamText(userMessages)`，`userMessages` 最后一条为当前输入。

### `StreamTextResult` + 回调类型

```ts
interface StreamTextCallbacks {
  onChunk?: (event: { chunk: TurnChunk }) => void;
  onStepFinish?: (event: { text: string; toolCalls: ToolCall[]; usage: Usage; step: number }) => void;
  onFinish?: (event: { text: string; usage: Usage; steps: number; toolCalls: ToolCall[] }) => void;
  onError?: (error: Error) => void;
}

interface StreamTextOptions extends ModelCallOptions {
  maxSteps?: number;
  onChunk?: ...;
  onStepFinish?: ...;
  onFinish?: ...;
  onError?: ...;
}

interface StreamTextResult {
  toDataStream(opts?: { headers?: Record<string, string> }): Response;
  toReadableStream(): ReadableStream<Uint8Array>;
  toAsyncIterable(): AsyncIterable<TurnChunk>;
  text: Promise<string>;
  usage: Promise<Usage>;
  on(event: "text-delta" | "tool-call-delta" | "tool-call-ready" | "done", fn: Function): () => void;
}
```

### 和 `run()` / `stream()` 对比

| | `run()` | `stream()` (旧) | `streamText()` (新) |
|---|---|---|---|
| 工具循环 | ✅ | ❌ | ✅ |
| 流式输出 | ❌ | ✅ | ✅ |
| 回调 | `onStep` | 无 | `onChunk` / `onStepFinish` / `onFinish` / `onError` |
| 首参 | `string` | `string` | `string` \| `ChatMessage[]` |
| 返回 | `Promise<TurnResult>` | `AsyncGenerator` | `StreamTextResult` |
| 消费 | `await` | `for await` | `.toDataStream()` 等 |

### 验收

| # | 标准 |
|---|---|
| F4.1 | `streamText("hello")` 正常流式输出文本 |
| F4.2 | `streamText(messages)` hydrate 历史，流式最后一条 |
| F4.3 | 工具自动执行，每步触发 `onStepFinish`，最后触发 `onFinish` |
| F4.4 | `maxSteps=2` 时工具循环在第 2 步后停止，`onFinish` 仍触发 |
| F4.5 | `toDataStream()` Response 被 AI SDK `useChat` 正确消费 |
| F4.6 | `toReadableStream()` / `toAsyncIterable()` 产生相同内容 |
| F4.7 | `stream.text` / `stream.usage` 在流结束后 resolve |
| F4.8 | `.on("tool-call-ready")` 事后钩子正常运行 |
| F4.9 | 旧 `agent.stream()` 行为不变 |
| F4.10 | `h.streamAgent()` 与手动 `agent.streamText()` 等价 |

## F5. 对象生成 → `generateObject()` + `streamObject()`

### 设计

Schema（Zod 或 JSON Schema）**不进 StablePrefix** — per-request 约束通过 `responseFormat` 透传 adapter。调用时忽略 tools（避免 OpenAI 冲突）。回复写入 MessageLog。

```ts
// Zod（类型推断 ✅）
const { object } = await agent.generateObject("Extract: Alice, 1992", {
  schema: z.object({ name: z.string(), year: z.number() }),
});
// object → { name: "Alice", year: 1992 } 类型安全

// JSON Schema（unknown）
const { object } = await agent.generateObject("...", {
  schema: { type: "object", properties: { name: { type: "string" } } },
});
// object → unknown

// 流式
for await (const chunk of agent.streamObject("Generate 10 items", {
  schema: z.object({ items: z.array(z.string()) }),
})) { /* partial object */ }
```

### 实现细节

- `generateObject` 内部调用 `adapter.chat(prefix, history, undefined, responseFormat, options)` — tools 传 `undefined`。
- Schema 转换：Zod → `zod-to-json-schema`（`{ target: "openAi" }` 兼容模式），JSON Schema → 透传。`responseFormat` 格式：`{ type: "json_schema", json_schema: { name: "output", schema: {...} } }`。
- 返回类型 `GenerateObjectResult<T>` 含 `object: T`（从 `JSON.parse(content)` 解析）+ `usage: Usage`。解析失败抛 `ObjectParseError`（含 raw content）。
- `streamObject` 返回 `AsyncGenerator<PartialObjectChunk<T>>`，内部用 `adapter.stream()` + 增量 JSON 解析（基于 `partial-json` 或手写状态机）。
- adapter 不强制要求 `responseFormat` 支持。若 adapter 不支持，`generateObject` 在 system prompt 末尾追加 `"\nRespond ONLY with valid JSON matching this schema: ..."` 作为 fallback（标记 `@degraded`）。

### 验收

| # | 标准 |
|---|---|
| F5.1 | Zod schema → `result.object` 具有编译时类型推断 |
| F5.2 | JSON Schema → `result.object` 为 `unknown` |
| F5.3 | `generateObject` 调用时 adapter 的 tools 参数为 `undefined` |
| F5.4 | JSON 解析失败抛 `ObjectParseError`，含 raw content |
| F5.5 | adapter 不支持 `responseFormat` 时走 system prompt fallback |
| F5.6 | `generateObject` 回复写入 MessageLog，后续 `run()` 可见 |
| F5.7 | `streamObject` 产生递增的 partial object |

## F6. 原生工具 + 默认 Skills

### 设计

- **不做独立 toolkit 包** — 代码量小，MCP 覆盖文件/网络。
- **不做内置 Cache-First Skills** — 未经验证，用户不启用则无存在感。
- **交付**：`calculator()` / `datetime()` 直接导出自 `@halo-sdk/core`。
- `discoverSkills` 脚手架 + 文档推荐社区 skill 清单。

```ts
import { calculator, datetime } from "@halo-sdk/core";
h.agent({ messages: [...], tools: { ...calculator(), ...datetime() } });
```

### 实现细节

- `calculator()`：`ToolDefinition<{ expression: string }>`，`execute` 用 `new Function("return " + expression)()` 安全求值（仅允许 `[0-9+\-*/().%\s]` 正则白名单，否则返回错误）。
- `datetime()`：`ToolDefinition<{ operation: string; value?: string; tz?: string }>`，支持 `"now"` / `"parse"` / `"convert"`。用 `Intl.DateTimeFormat`，零依赖。

### 验收

| # | 标准 |
|---|---|
| F6.1 | `calculator("2 + 3 * 4")` → `"14"` |
| F6.2 | `calculator("process.exit()")` → 错误（白名单拒绝） |
| F6.3 | `datetime({ operation: "now", tz: "Asia/Shanghai" })` 返回正确时间 |

## Adapter 变更

```ts
interface ModelAdapter {
  chat(
    prefix: ChatMessage[],
    history: ChatMessage[],
    tools?: ToolSpec[],
    responseFormat?: ResponseFormat,    // 新增
    options?: ModelCallOptions,         // 新增
  ): Promise<{ content: string; toolCalls: ToolCall[]; usage: Usage }>;

  stream(
    prefix: ChatMessage[],
    history: ChatMessage[],
    tools?: ToolSpec[],
    responseFormat?: ResponseFormat,    // 新增
    options?: ModelCallOptions,         // 新增
  ): AsyncGenerator<TurnChunk>;
}
```

`DeepSeekAdapter` 同步更新：`responseFormat` 映射到 `body.response_format`，`options` 映射到 `body.temperature` / `body.top_p` / `body.max_tokens`。
