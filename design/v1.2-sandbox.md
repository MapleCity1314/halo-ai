# Halo SDK v1.2 — Sandbox 设计

> 状态：方案确认，可实现 | 更新：2025-07-16

## 原则

- **对缓存模型零侵入** — sandbox 不进 StablePrefix，不进缓存指纹
- **跨平台** — VirtualSandbox 覆盖浏览器/Edge，ContainerSandbox 仅 Node
- **安全默认** — 路径硬隔离 + 结构化命令 + 多层清理
- **向后兼容** — `execute(args, ctx)` 运行时检测 `length`，旧代码零影响

---

## 一、分层设计

```
Sandbox (interface)
├── VirtualSandbox       纯内存 Map 实现，跨平台，无 exec
└── ContainerSandbox     真实 Node fs + child_process
```

| | VirtualSandbox | ContainerSandbox |
|---|---|---|
| readFile / writeFile / readdir | ✅ | ✅ |
| exec (命令执行) | ❌ | ✅ |
| 持久化 | ❌ 内存，会话完消失 | ✅ 落盘 |
| 目标运行时 | 浏览器 / Edge / 测试 | Node.js |
| 核心场景 | Skills 加载、临时工作区 | 代码生成→执行→验证、Evals |

VirtualSandbox 不是"弱化版 ContainerSandbox" — 它是文件系统的浏览器 polyfill，让 Skills 在非 Node 运行时也能活。

## 二、Sandbox 接口

```ts
interface Sandbox {
  // ── 文件系统（所有后端必须实现）──
  readFile(path: string): Promise<string>;
  writeFile(path: string, content: string): Promise<void>;
  readdir(path: string): Promise<{ name: string; isDirectory: boolean }[]>;
  exists(path: string): Promise<boolean>;

  // ── 命令执行（仅 ContainerSandbox）──
  exec?(command: string, args: string[], opts?: ExecOptions): Promise<ExecResult>;
  execShell?(command: string, opts?: ExecShellOptions): Promise<ExecResult>;

  // ── 生命周期 ──
  /** Kill all background processes, remove temp files. Idempotent. */
  dispose(): Promise<void>;

  // ── 查询 ──
  readonly cwd: string;
  readonly hasExec: boolean;

  /**
   * Auto-generate tool definitions matching this sandbox’s capabilities.
   * VirtualSandbox: { readFile, writeFile, readdir }
   * ContainerSandbox: above + { exec, execShell, httpGet }
   */
  tools(): Record<string, ToolDefinition>;
}
```

## 三、VirtualSandbox 实现细节

```ts
class VirtualSandbox implements Sandbox {
  readonly cwd = "/sandbox";
  readonly hasExec = false;
  private _files = new Map<string, string>();   // path → content
  private _dirs  = new Set<string>();            // directory paths

  async readFile(path: string): Promise<string> {
    const normalized = this._normalize(path);
    const content = this._files.get(normalized);
    if (content === undefined) throw new SandboxError(`ENOENT: ${path}`, "ENOENT");
    return content;
  }

  async writeFile(path: string, content: string): Promise<void> {
    const normalized = this._normalize(path);
    // Auto-create parent directories.
    const parent = normalized.slice(0, normalized.lastIndexOf("/"));
    if (parent) this._ensureDir(parent);
    this._files.set(normalized, content);
  }

  async readdir(path: string): Promise<{ name: string; isDirectory: boolean }[]> { ... }
  async exists(path: string): Promise<boolean> { ... }
  async dispose(): Promise<void> { this._files.clear(); this._dirs.clear(); }

  private _normalize(p: string): string {
    // Resolve . and .. against cwd. No symlink resolution (virtual fs).
    const resolved = pathPosix.resolve(this.cwd, p);
    if (!resolved.startsWith(this.cwd)) throw new SandboxError("EACCES", "EACCES");
    return resolved;
  }
}
```

**关键决策：** VirtualSandbox 用纯 `Map` — 零依赖、序列化友好（`JSON.stringify([..._files])` 即可保存/恢复状态）。

## 四、ContainerSandbox 实现细节

```ts
class ContainerSandbox implements Sandbox {
  readonly cwd: string;
  readonly hasExec = true;
  private _baseDir: string;
  private _processes = new Map<number, ProcessHandle>();
  private _config: ContainerConfig;

  constructor(opts: ContainerConfig) {
    this._baseDir = path.resolve(opts.baseDir);
    this.cwd = this._baseDir;
    this._config = opts;
  }

  // ── 路径硬隔离 ──
  private _resolve(p: string): string {
    const resolved = path.resolve(this._baseDir, p);
    // Resolve symlinks and compare real path.
    const realPath = fs.realpathSync(resolved);
    if (!realPath.startsWith(this._baseDir + path.sep) &&
        realPath !== this._baseDir) {
      throw new SandboxError(`Path traversal blocked: ${p}`, "EACCES");
    }
    return resolved;
  }

  // ── 文件操作 ──
  async readFile(p: string): Promise<string> {
    return fs.readFile(this._resolve(p), "utf-8");
  }
  async writeFile(p: string, content: string): Promise<void> {
    const target = this._resolve(p);
    await fs.mkdir(path.dirname(target), { recursive: true });
    await fs.writeFile(target, content, "utf-8");
  }

  // ── 命令执行 ──
  async exec(command: string, args: string[], opts?: ExecOptions): Promise<ExecResult> {
    const timeout = opts?.timeout ?? this._config.execTimeout ?? 30_000;
    const cwd = opts?.cwd ? this._resolve(opts.cwd) : this.cwd;

    return new Promise((resolve, reject) => {
      const child = spawn(command, args, {
        cwd,
        env: { ...process.env, ...opts?.env },
        timeout,
        signal: AbortSignal.timeout(timeout),
      });

      let stdout = "";
      let stderr = "";
      child.stdout.on("data", (d: Buffer) => stdout += d.toString());
      child.stderr.on("data", (d: Buffer) => stderr += d.toString());

      child.on("close", (code) => resolve({ stdout, stderr, exitCode: code ?? 1 }));
      child.on("error", reject);
    });
  }

  // ── 资源限制 ──
  // CPU/memory: Linux cgroup (via cgroupfs write); non-Linux → noop warning
  // Disk: du -sh on writeFile; reject if over limit
  // Background processes: maxBackgroundProcesses enforced in exec()

  // ── 清理 ──
  async dispose(): Promise<void> {
    for (const [pid, handle] of this._processes) {
      try { handle.kill("SIGKILL"); } catch { /* already dead */ }
    }
    this._processes.clear();
  }

  [Symbol.dispose](): void { void this.dispose(); }
}
```

### ContainerConfig

```ts
interface ContainerConfig {
  baseDir: string;                  // 必填 — sandbox 根目录
  env?: Record<string, string>;     // 注入到子进程的环境变量
  execTimeout?: number;             // 默认 30s
  cpuLimit?: string;                // Linux cgroup，如 "50%"
  memoryLimit?: string;             // Linux cgroup，如 "512MB"
  maxDiskUsage?: string;            // 如 "100MB"
  maxBackgroundProcesses?: number;  // 默认 3
  allowOutbound?: boolean;          // 子进程外网访问，默认 true
  allowedHosts?: string[];          // 外网白名单
}
```

## 五、命令执行

### ExecOptions

```ts
interface ExecOptions {
  cwd?: string;               // 相对于 baseDir
  env?: Record<string, string>;
  timeout?: number;           // 覆盖实例默认值
  background?: boolean;       // false: 等退出；true: 返回 ProcessHandle
  exposePort?: number;        // 后台进程暴露端口（0 = 自动分配）
  bindHost?: string;          // 默认 "127.0.0.1"
  portTTL?: number;           // 无活动自动 kill，默认 300_000 (5min)
  portAuth?: { type: "token"; value: string };
}

interface ExecShellOptions extends ExecOptions {
  allowShell: boolean;        // 必须显式 true
}

interface ExecResult {
  stdout: string;
  stderr: string;
  exitCode: number;
}

interface ProcessHandle {
  pid: number;
  url?: string;               // 仅 exposePort 时
  stdout: AsyncIterable<string>;
  stderr: AsyncIterable<string>;
  kill(signal?: string): Promise<void>;
  wait(): Promise<ExecResult>;
}
```

### 端口安全

- 默认 bind `127.0.0.1`，仅 sandbox 和 localhost 可访问
- 可选 `portAuth.token` — URL 参数验证
- TTL 自动 kill — `setTimeout(ttl, kill)` + 每次 httpGet 刷新 TTL

## 六、`tools()` 自动收窄

```ts
// VirtualSandbox.tools() → file tools only
{
  readFile:  ToolDefinition<{ path: string }>;
  writeFile: ToolDefinition<{ path: string; content: string }>;
  readdir:   ToolDefinition<{ path: string }>;
}

// ContainerSandbox.tools() → file + exec + httpGet
{
  readFile:  ToolDefinition<...>;
  writeFile: ToolDefinition<...>;
  readdir:   ToolDefinition<...>;
  exec:      ToolDefinition<{ command: string; args: string[]; ... }>;
  execShell: ToolDefinition<{ command: string; ... }>;
  httpGet:   ToolDefinition<{ url: string; headers?: Record<string,string> }>;
}
```

`httpGet` 工具仅在有 `exposePort` 的后台进程时有用 — 它知道 sandbox 的端口映射，agent 可通过它验证服务器。

## 七、Agent 集成 & `execute` 签名变更

### 注入点

```ts
const agent = h.agent({
  sandbox: new ContainerSandbox({ baseDir: "/tmp/work" }),
  skills,
  // sandbox.tools() spread 到 tools
  tools: { ...sandbox.tools(), ...myCustomTools },
});
```

### ToolContext

```ts
interface ToolContext {
  sandbox: Sandbox;
}
```

### `execute` 向后兼容

```ts
// TypeScript overload:
export function tool<TArgs>(
  def: ToolDefinition<TArgs> & {
    execute: (args: TArgs) => string | Promise<string>;
  },
): ToolDefinition<TArgs>;
export function tool<TArgs>(
  def: ToolDefinition<TArgs> & {
    execute: (args: TArgs, ctx: ToolContext) => string | Promise<string>;
  },
): ToolDefinition<TArgs>;

// Runtime:
const paramCount = executor.length; // 不含默认参数
const result = paramCount >= 2
  ? await executor(parsed, { sandbox })
  : await executor(parsed);
```

旧工具 `execute: (args) => ...` 不传 ctx，零影响。新工具 `execute: (args, ctx) => ...` 拿到 sandbox。

## 九、缓存模型（零侵入确认）

```
StablePrefix (CACHED)              MessageLog (UNCACHED)         进程外
─────────────────────              ─────────────────────          ──────────
system prompt                      user messages                 ContainerSandbox
skills name+description             assistant messages             ├── baseDir 文件
sandbox.tools() 的 ToolSpec         tool results                  ├── 子进程
model config                        sandbox 产出的内容             └── 端口映射

sandbox 对象引用 → 不在 prefix
sandbox 文件内容 → 不在 prefix
后台进程状态 → 完全独立
```

## 十、性能

| | VirtualSandbox | ContainerSandbox |
|---|---|---|
| 构造 | <1ms | ~3ms |
| 内存占用 | ~2MB（Map） | ~5MB + 子进程 |
| readFile/writeFile | <0.1ms（内存） | ~0.5ms（磁盘） |
| exec 冷启动 | N/A | ~50ms（spawn） |
| 生命周期建议 | 每次 agent 调用新建 | 模块级别复用单例 |

## 十一、验收标准

| # | 标准 |
|---|---|
| S1 | VirtualSandbox: readFile/writeFile/readdir 正确执行 |
| S2 | ContainerSandbox: `../../etc/passwd` 被拒绝 |
| S3 | ContainerSandbox: 符号链接指向 baseDir 外部被拒绝 |
| S4 | ContainerSandbox: `exec("echo", ["hello"])` → stdout "hello\n" |
| S5 | ContainerSandbox: `execShell("echo hello", { allowShell: true })` → stdout "hello\n" |
| S6 | ContainerSandbox: 后台进程 exposePort → url 正确返回 |
| S7 | ContainerSandbox: 端口 TTL 到期自动 kill |
| S8 | VirtualSandbox.tools() 不含 exec/execShell/httpGet |
| S9 | ContainerSandbox.tools() 含 exec/execShell/httpGet |
| S10 | `execute(args)` 不传 ctx — 向后兼容 |
| S11 | `execute(args, ctx)` 收到 ctx.sandbox |
| S12 | `discoverSkills({ sandbox })` 正常工作 |
| S13 | `agent.clearLog()` 不清理 sandbox 文件 |
| S14 | sandbox 对象引用不进 StablePrefix fingerprint |
| S15 | `sandbox.dispose()` 杀掉所有子进程 |
| S16 | `ContainerSandbox[Symbol.dispose]()` 自动清理 |
