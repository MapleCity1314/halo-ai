# Halo SDK v1.2 — Sandbox 设计

> 状态：方案讨论完成，待实现
> 最后更新：2025-07-16

## 概述

Sandbox 为 Halo Agent 提供**文件系统抽象**和**命令执行能力**。v1.1 遗留的 `@nodeOnly` 缺口（`discoverSkills`、`loadSkill` 硬编码 Node `fs`）由 sandbox 填补。

核心原则：
- **对缓存模型零侵入** — sandbox 不进 StablePrefix，不进缓存指纹
- **跨平台** — VirtualSandbox 覆盖浏览器/Edge，ContainerSandbox 仅 Node
- **安全默认** — 路径硬隔离 + 结构化命令 + 多层清理

---

## 一、分层设计

```
Sandbox (interface)           统一契约
├── VirtualSandbox             纯内存，跨平台
└── ContainerSandbox           真实 fs + shell
```

| | VirtualSandbox | ContainerSandbox |
|---|---|---|
| readFile / writeFile / readdir | ✅ | ✅ |
| exec (命令执行) | ❌ 物理不可能 | ✅ |
| 持久化 | ❌ 会话结束消失 | ✅ 落盘 |
| 目标运行时 | 浏览器 / Edge / 测试 | Node.js |
| 核心场景 | Skills 加载、临时文件 | 代码生成 → 执行 → 验证 |

VirtualSandbox 不是"弱化版 ContainerSandbox" — 它是**文件系统的浏览器 polyfill**，让 Skills 在非 Node 运行时也能工作。

## 二、接口设计

### Sandbox

```ts
interface Sandbox {
  // ── 文件系统（所有后端必须实现）──
  readFile(path: string): Promise<string>;
  writeFile(path: string, content: string): Promise<void>;
  readdir(path: string): Promise<{ name: string; isDirectory: boolean }[]>;

  // ── 命令执行（ContainerSandbox 实现；VirtualSandbox 上不存在）──
  exec?(
    command: string,
    args: string[],
    opts?: ExecOptions,
  ): Promise<ExecResult>;

  execShell?(
    command: string,
    opts?: ExecShellOptions,
  ): Promise<ExecResult>;

  // ── 查询 ──
  readonly cwd: string;
  readonly hasExec: boolean;

  /** 自动生成适配当前 sandbox 类型的工具集。 */
  tools(): ToolsForSandbox;
}
```

### ExecOptions

```ts
interface ExecOptions {
  cwd?: string;
  env?: Record<string, string>;
  timeout?: number;           // 默认 30s
  background?: boolean;       // false: 等退出；true: 返回 ProcessHandle
  exposePort?: number;        // 后台进程暴露端口（0 = 自动分配）
}

interface ProcessHandle {
  pid: number;
  url?: string;               // 仅 exposePort 时
  stdout: AsyncIterable<string>;
  stderr: AsyncIterable<string>;
  kill(signal?: string): Promise<void>;
  wait(): Promise<ExecResult>;
}

interface ExecResult {
  stdout: string;
  stderr: string;
  exitCode: number;
}
```

### `tools()` — 自动收窄

```ts
// VirtualSandbox.tools() → 仅文件工具
tools(): {
  readFile: ToolDefinition<{ path: string }>;
  writeFile: ToolDefinition<{ path: string; content: string }>;
  readdir: ToolDefinition<{ path: string }>;
}

// ContainerSandbox.tools() → 文件 + exec 全家
tools(): {
  readFile: ToolDefinition<{ path: string }>;
  writeFile: ToolDefinition<{ path: string; content: string }>;
  readdir: ToolDefinition<{ path: string }>;
  exec: ToolDefinition<{ command: string; args: string[]; ... }>;
  execShell: ToolDefinition<{ command: string; ... }>;
  httpGet: ToolDefinition<{ url: string; headers?: Record<string, string> }>;
}
```

Agent 不会试图调用不存在的工具 — 工具列表里根本没有。

## 三、安全性

### 路径硬隔离

```ts
class ContainerSandbox {
  private _resolve(path: string): string {
    const resolved = pathModule.resolve(this._baseDir, path);
    const realPath = fs.realpathSync(resolved);
    if (!realPath.startsWith(this._baseDir)) {
      throw new SandboxError(`Path traversal blocked: ${path}`);
    }
    return resolved;
  }
}
```

| 攻击向量 | 防御 |
|---|---|
| `../../etc/passwd` | `resolve()` 规范化 + startsWith |
| 符号链接指向外部 | `realpathSync` 解析真实路径 |
| `/absolute/path` 绕过 | 视为 baseDir 相对路径，规范化后仍在 jail |
| Windows `..\..\` | `resolve()` 跨平台规范化 |

### 命令安全

- `exec(command, args)` — 结构化 argv，不走 shell（对标 Docker API）
- `execShell(command, opts)` — shell 字符串，需 `allowShell: true` opt-in

```ts
// ✅ 安全：结构化参数
sandbox.exec("python", ["fib.py"]);

// ⚠️ opt-in：明确允许 shell 解释
sandbox.execShell("python fib.py 2>&1 | tee output.txt", { allowShell: true });
```

### 资源限制

```ts
new ContainerSandbox({
  baseDir: "/tmp/workspace",
  cpuLimit: "50%",             // cgroup cpu.cfs_quota_us (Linux)
  memoryLimit: "512MB",        // cgroup memory.limit_in_bytes (Linux)
  execTimeout: 30_000,         // 单次超时
  maxBackgroundProcesses: 3,
  maxDiskUsage: "100MB",
});
```

| 资源 | Linux | 非 Linux 降级 |
|---|---|---|
| CPU | cgroup | worker_threads 时间片 |
| 内存 | cgroup `memory.max` | `process.memoryUsage` 轮询 + kill |
| 超时 | `AbortController` + `subprocess.kill()` | 通用 |
| 磁盘 | `du -sh` 前置检查 | 通用 |

### 端口安全

```ts
sandbox.exec("node", ["server.js"], {
  background: true,
  exposePort: 0,               // 0 = 自动分配
  bindHost: "127.0.0.1",       // 默认仅本地回环
  portTTL: 300_000,            // 5 分钟无活动自动 kill
  portAuth: { type: "token", value: crypto.randomUUID() },
});
```

- 默认 bind 127.0.0.1，仅 sandbox 和 localhost 可访问
- 可选 token 认证
- TTL 自动回收

### 进程清理

| 层级 | 机制 |
|---|---|
| Agent 主动 | `sandbox.stopAll()` / `ProcessHandle.kill()` |
| Sandbox 析构 | `Symbol.dispose` 自动 SIGKILL |
| TTL 兜底 | 后台进程 5 分钟无活动自动 kill |
| 系统兜底 | `process.on('exit', cleanup)` |

## 四、集成模式

### Agent 注入

```ts
const sandbox = new ContainerSandbox({ baseDir: "/home/project" });

const agent = h.agent({
  messages: [{ role: "system", content: "..." }],
  sandbox,
  skills,                              // ← skills 自动用 sandbox 读文件
  tools: {
    ...sandbox.tools(),               // ← 根据 sandbox 类型自动收窄
  },
});
```

### 用户工具接收 sandbox

```ts
// execute 签名从 (args) 扩展为 (args, ctx)
const myTool = tool({
  description: "Read a config file",
  parameters: { ... },
  execute: async (args, ctx) => {
    // ctx.sandbox — 由 agent 自动注入
    const content = await ctx.sandbox.readFile(args.path);
    return `Config: ${content}`;
  },
});
```

向后兼容：运行时检测 `execute.length`。1 参数 → 不传 ctx。用户代码零影响。

```ts
interface ToolContext {
  sandbox: Sandbox;
}
```

### Skills 集成

```ts
// discoverSkills 接受可选 sandbox
const skills = await discoverSkills({
  directories: [".halo/skills"],
  sandbox,                             // ← 不再硬编码 Node fs
});

// loadSkill executor 从 agent sandbox 读文件
// agent 内部自动完成，用户无感
```

## 五、与 MCP 的关系

```
Sandbox (本地)                  MCP Server (远程)
─────────────────────          ─────────────────────
agent 内部文件访问             第三方能力（文件系统、数据库、API）
readFile / writeFile           tools/list → ToolDefinition
exec / execShell               tools/call → 远程执行
进程内零开销                   子进程隔离
```

**互补，不替代。** Sandbox 管本地轻量操作，MCP 管外部能力。

## 六、缓存模型（零侵入）

```
StablePrefix (CACHED)             MessageLog (UNCACHED)          进程外
─────────────────────             ─────────────────────           ──────────
system prompt                     user: "write fib.py"            ContainerSandbox
skills 元数据                     assistant: "ok"                 ├── /tmp/workspace/
sandbox tools 定义                  tool: writeFile result          │   ├── fib.py
model config                       user: "run it"                 │   └── config.json
                                   tool: exec result              └── server.js (pid 42)
                                   assistant: "fib(20)=6765"          port 52031 → URL

sandbox 对象引用不在 prefix 里
文件内容不在 prefix 里
进程状态完全独立
```

## 七、性能

| | VirtualSandbox | ContainerSandbox |
|---|---|---|
| 创建耗时 | ~1ms | ~3ms |
| 内存占用 | ~2MB | ~5MB + 子进程 |
| exec 冷启动 | N/A | ~50ms (spawn) |
| 生命周期建议 | 每次 agent 调用新建 | 复用单例 |

```ts
// ContainerSandbox 适合长生命周期
const sandbox = new ContainerSandbox({ baseDir: "/home/project" });

for (const task of tasks) {
  const agent = h.agent({ sandbox, ... });  // 复用
  await agent.run(task);
}

// VirtualSandbox 可每次新建（零成本）
const agent = h.agent({
  sandbox: new VirtualSandbox(),
});
```

## 八、典型场景

### 代码生成 → 执行 → 验证

```ts
const agent = h.agent({
  sandbox: new ContainerSandbox({ baseDir: "/tmp/work" }),
  tools: { ...sandbox.tools() },
});

await agent.run(
  "Write a Python script that calculates fibonacci(30), then run it and report the result"
);
// Agent 自动：writeFile("fib.py") → exec("python", ["fib.py"]) → 读 stdout → 回复结果
```

### Skills 跨平台加载

```ts
// Node.js
const sandbox = new ContainerSandbox({ baseDir: process.cwd() });
const skills = await discoverSkills({ directories: [".halo/skills"], sandbox });

// Edge Runtime / 浏览器
const sandbox = new VirtualSandbox();
// 手动注册文件内容
await sandbox.writeFile("/skills/my-skill/SKILL.md", skillContent);
const skills = [{ name: "my-skill", description: "...", path: "/skills/my-skill" }];
```

### Evals — 运行服务器并测试

```ts
const sandbox = new ContainerSandbox({ baseDir: "/tmp/eval" });

const agent = h.agent({ sandbox, tools: { ...sandbox.tools() } });

// 1. 搭建项目
await agent.run("Create an Express server with /health endpoint");

// 2. 启动后台进程
await agent.run("Start the server on a free port, then test /health returns 200");

// 3. 应用层展示端口
const process = sandbox.getProcessByPort(52031);
return NextResponse.json({ url: process!.url });
// → "http://localhost:52031"
```

## 九、验收标准

| # | 标准 |
|---|---|
| S1 | VirtualSandbox: readFile/writeFile/readdir 正确执行 |
| S2 | ContainerSandbox: 路径 `../../etc/passwd` 被拦截 |
| S3 | ContainerSandbox: exec("echo", ["hello"]) → stdout "hello" |
| S4 | ContainerSandbox: 后台进程 ToolDefinition 正确返回 ProcessHandle |
| S5 | Sandbox tools 自动收窄：VirtualSandbox 不含 exec |
| S6 | execute(args, ctx) 和 execute(args) 向后兼容 |
| S7 | discoverSkills({ sandbox }) 正常工作 |
| S8 | agent.clearLog() 不清理 sandbox 文件 |
| S9 | sandbox 对象引用不进 StablePrefix |
| S10 | 符号链接指向外部目录被拒绝 |
